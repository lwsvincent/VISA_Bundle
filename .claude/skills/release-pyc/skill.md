# Skill: release-pyc

Build a `cp311-cp311-win_amd64` wheel where `visa_bundle` implementation `.py`
source files are compiled to `.pyc` bytecode and removed, protecting IP
while preserving IDE type hints through `.pyi` stub files.

**Protected layers (compiled to .pyc):**
- Root package modules -- `Setting.py`, `VISA.py` -> `.pyc` + stubgen generates `.pyi`

**Preserved as `.py` (public API):**
- All `__init__.py` files

## Pre-flight Check

Before running the build, verify:
1. `.venv\Scripts\python.exe` exists -- if not, run `call template_project\scripts\create_venv.bat -local -dev`
2. `setup.py` exists at project root
3. `pyproject.toml` contains `__pycache__/*.pyc` globs in package-data

If any check fails, report the issue and stop.

## Run the Build

Execute:
```
call scripts\build_pyc_wheel.bat
```

Watch the output for:
- `[ERROR]` lines -- fatal, stop and diagnose
- `[WARNING]` lines -- note but continue (usually stubgen import issues)
- `Build SUCCESS:` line -- confirms the wheel path

## Verify Results

After a successful build, confirm the wheel contains:

**Must have `.pyc`:**
- `visa_bundle/__pycache__/Setting.cpython-311.pyc`
- `visa_bundle/__pycache__/VISA.cpython-311.pyc`

**Must have `.pyi`:**
- `visa_bundle/Setting.pyi` (generated by stubgen)
- `visa_bundle/VISA.pyi`

**Must NOT have bare `.py`** (only `__init__.py` permitted):
- `visa_bundle/Setting.py` -- should be absent
- `visa_bundle/VISA.py` -- should be absent

## Error Diagnosis

| Error | Cause | Fix |
|---|---|---|
| `Virtual environment not found` | `.venv` missing | `call template_project\scripts\create_venv.bat -local -dev` |
| `Failed to install mypy` | Network / pip issue | Check internet, `pip install mypy` manually |
| `No .pyc files found in wheel` | Missing pyc globs in package-data | Check `pyproject.toml` has `__pycache__/*.pyc` globs |
| `setup.py` not found | setup.py missing | Verify `setup.py` exists at project root |
| `import wheel` fails | wheel not installed | `.venv\Scripts\pip install wheel` |

## Report to User

After the build, summarize:
- Wheel file name and path (e.g., `dist/visa_bundle-1.0.0-cp311-cp311-win_amd64.whl`)
- All verification checks passed/failed
- Any warnings encountered

## Optional: Local Test

To verify the wheel installs and tests pass:
```
.venv\Scripts\pip install dist\<wheel-name>.whl --force-reinstall
call template_project\scripts\run_tests.bat
```

## Notes

- This wheel is **Python 3.11 only** -- will not install on 3.10 or 3.12
- Source files in `src/` are **never modified** -- only the build directory is affected
- Running the script again is safe and idempotent
